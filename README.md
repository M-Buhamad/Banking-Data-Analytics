Banking Data Analytics Project
Overview

This project focuses on analyzing banking data to uncover customer behavior, transaction trends, and potential risk patterns.
The workflow combines data preprocessing using Python and data visualization using Power BI to deliver actionable insights.

Dataset

The analysis is based on three main datasets:

Users: Customer demographic information
Cards: Card details linked to each user
Transactions: Financial transactions data
Tools & Technologies
Python (Pandas) → Data cleaning and preprocessing
Power BI → Data modeling and dashboard creation
Excel → Initial data storage
Workflow
Data Cleaning
Handling missing values
Standardizing formats
Preparing datasets for analysis
Data Modeling
Creating relationships between Users, Cards, and Transactions
Building a structured data model
Visualization
Designing interactive dashboards
Enabling filtering by demographics and transaction attributes
 Analysis Highlights
Customer distribution and card ownership trends
Transaction behavior across different channels (Online vs Branch)
Failed transaction patterns and error types
Financial behavior segmented by age group and income
Key Insights
Online transactions represent the majority of total activity
Transaction failure rates vary significantly across age groups
Higher income segments show more stable transaction behavior
Certain error types (e.g., insufficient balance, incorrect PIN) are dominant
Future Improvements
Apply Machine Learning models to detect fraudulent transactions
Build predictive models for customer behavior
Automate data pipelines for real-time analytics
